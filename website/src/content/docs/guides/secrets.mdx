This guide covers the two shapes for getting a value out of `.env`
(or any [`ConfigProvider`](https://effect.website/docs/configuration))
and onto a deploy target as an env binding. Which one you use depends on
**how the Worker is written**, not what kind of value it is:

| Worker shape                                          | Use                                                     |
| ----------------------------------------------------- | ------------------------------------------------------- |
| **Effect Worker** — `Cloudflare.Worker(…, Effect.gen(…))` | `yield* Alchemy.Secret(name)` / `yield* Alchemy.Variable(name)` |
| **Async Worker** — `Cloudflare.Worker(…, { env: {…} })`   | `env: { KEY: yield* Config.string("KEY") }` (in the Stack) |
| **Vite / static-asset app** — `Cloudflare.Vite(…)`        | same as async — pass resolved values into `env`         |

`Alchemy.Secret` / `Alchemy.Variable` are **only** valid inside an
Effect Worker's init phase, because they return a runtime accessor
the Worker `yield*`s inside `fetch`. An async Worker has no init
phase — it's a class with an `async fetch(req, env)` method — so
there's nothing to attach an accessor to. Pre-resolve the value in
the Stack and drop it into `env` instead.

## Add the env var

Put the value somewhere the active `ConfigProvider` can find it.
The CLI uses `ConfigProvider.fromEnv()` by default, so either an
exported shell var or a `.env` file at the project root works:

```sh
# .env
OPENAI_API_KEY=sk-proj-...
VITE_APP_ENV=dev
```

You don't need to plumb `process.env` anywhere — every approach
below reads from `ConfigProvider`.

## Effect Worker

Inside the Worker's init phase, `yield* Alchemy.Secret("OPENAI_API_KEY")`
records the binding and returns an `Effect<Redacted<string>>` —
an **accessor**. `yield*` it inside `fetch` to read the value:

```typescript
// src/worker.ts
import * as Alchemy from "alchemy";
import * as Cloudflare from "alchemy/Cloudflare";
import * as Effect from "effect/Effect";
import * as Redacted from "effect/Redacted";
import * as HttpServerResponse from "effect/unstable/http/HttpServerResponse";

export default Cloudflare.Worker(
  "Worker",
  { main: import.meta.path },
  Effect.gen(function* () {
    const apiKey = yield* Alchemy.Secret("OPENAI_API_KEY");
    const appEnv = yield* Alchemy.Variable("VITE_APP_ENV");

    return {
      fetch: Effect.gen(function* () {
        const key = yield* apiKey;   // Redacted<string>
        const env = yield* appEnv;   // string

        const res = yield* Effect.tryPromise(() =>
          fetch("https://api.openai.com/v1/models", {
            headers: { Authorization: `Bearer ${Redacted.value(key)}` },
          }),
        );
        return HttpServerResponse.text(`${env}: ${res.status}`);
      }),
    };
  }),
);
```

`Alchemy.Secret("X")` is sugar for `Alchemy.Secret("X", Config.redacted("X"))`;
`Alchemy.Variable("X")` is sugar for `Alchemy.Variable("X", Config.string("X"))`.
At deploy time Alchemy resolves the value, ships it to the Worker
as a `secret_text` / `plain_text` binding, and `Redacted` keeps
the secret out of logs and stack traces.

## Async Worker

An async Worker is just a class with `async fetch(request, env)`.
There's no Init `Effect.gen` to `yield*` `Alchemy.Secret` inside —
the `env` object on the Worker definition takes pre-resolved
values directly:

```typescript
// alchemy.run.ts
import * as Alchemy from "alchemy";
import * as Cloudflare from "alchemy/Cloudflare";
import * as Config from "effect/Config";
import * as Effect from "effect/Effect";

export default Alchemy.Stack(
  "App",
  { providers: Cloudflare.providers(), state: Cloudflare.state() },
  Effect.gen(function* () {
    const worker = yield* Cloudflare.Worker("Worker", {
      main: "./src/worker.ts",
      env: {
        OPENAI_API_KEY: yield* Config.redacted("OPENAI_API_KEY"),
        VITE_APP_ENV:   yield* Config.string("VITE_APP_ENV"),
      },
    });

    return worker.url;
  }),
);
```

Read them inside the async handler the normal Cloudflare way:

```typescript
// src/worker.ts
import type { WorkerEnv } from "../alchemy.run.ts";

export default {
  async fetch(request: Request, env: WorkerEnv) {
    const res = await fetch("https://api.openai.com/v1/models", {
      headers: { Authorization: `Bearer ${env.OPENAI_API_KEY}` },
    });
    return new Response(`${env.VITE_APP_ENV}: ${res.status}`);
  },
};
```

:::caution
Every value in `env: { … }` must already be a resolved `string`
or `Redacted<string>`. You can't drop a `Config` in there — the
worker provider sees `Config<…>` as a JSON object and ships it as
a literal. Always `yield*` first:

```diff lang="typescript"
  env: {
-   VITE_APP_ENV: Config.string("VITE_APP_ENV"),
+   VITE_APP_ENV: yield* Config.string("VITE_APP_ENV"),
  },
```

If one entry needs `yield*`, the whole `env` block needs `yield*`
on each entry. Mixing resolved strings with raw `Config` values
ships the raw ones as JSON.
:::

## Vite / static-asset apps

`Cloudflare.Vite` (and any other static-asset shape) has the same
`env: { … }` contract as an async Worker. `VITE_*` vars are baked
into the client bundle at build time, so pre-resolve them in the
Stack and pass strings:

```typescript
import * as Alchemy from "alchemy";
import * as Cloudflare from "alchemy/Cloudflare";
import * as Config from "effect/Config";
import * as Effect from "effect/Effect";

export default Alchemy.Stack(
  "Website",
  { providers: Cloudflare.providers(), state: Cloudflare.state() },
  Effect.gen(function* () {
    const site = yield* Cloudflare.Vite("Website", {
      env: {
        VITE_APP_ENV: yield* Config.string("VITE_APP_ENV"),
        VITE_API_URL: yield* Config.string("VITE_API_URL"),
      },
    });

    return site.url;
  }),
);
```

`Alchemy.Secret` / `Alchemy.Variable` aren't an option here —
there's no Worker init phase running on Cloudflare's edge to
resolve them. Everything has to be baked in at deploy time.

## Override the source

The 2-arg form of both helpers replaces the default `Config` with
whatever you want. Useful when the on-disk env var has a different
name, or the value comes from another resource:

```typescript
// Different env-var name
yield* Alchemy.Secret("OPENAI_API_KEY", Config.redacted("MY_OPENAI_KEY"));

// Mint a random secret on first deploy, persist in state
const random = yield* Alchemy.Random("SESSION_SECRET");
yield* Alchemy.Secret("SESSION_SECRET", random.text);

// Pipe a Cloudflare account-level secret into the Worker binding
const stored = yield* Cloudflare.Secret("ApiKey", { store, value: … });
yield* Alchemy.Secret("OPENAI_API_KEY", stored.value);
```

The first argument is always the binding name (the key the runtime
reads). The second is just where the value comes from.

## Recap

- **Effect Worker** → `yield* Alchemy.Secret/Variable(name)` in
  init, `yield*` the accessor in `fetch`. Typed runtime access,
  `Redacted` keeps secrets out of logs.
- **Async Worker / Vite** → no init phase. `yield* Config.string/redacted(name)`
  inside the Stack's `Effect.gen` and put the resolved value in
  `env: { … }`. Every entry must be resolved — `Config` objects
  don't ride.
- Both routes read from the same `.env`/`ConfigProvider`; the
  binding on the deploy target ends up the same shape.

For the bigger picture — input shapes, the deploy-vs-runtime
phases, and when to reach for Cloudflare's Secrets Store instead —
see [Concepts › Secrets and Variables](/concepts/secrets).
