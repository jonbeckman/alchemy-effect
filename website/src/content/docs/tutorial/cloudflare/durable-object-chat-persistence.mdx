Effect AI's `Persistence` module is how the chat primitives keep a
conversation's message history around between calls. By default the
chat store lives in memory, which is fine for in-process demos but
useless across a Worker fleet.
`Cloudflare.DurableObjectChatPersistence` is a `BackingPersistence`
layer that uses the surrounding DO's `state.storage` instead.

## Define a chat-backing DO

```typescript
// src/chat-backend.ts
import * as Cloudflare from "alchemy/Cloudflare";
import * as Effect from "effect/Effect";

export default class ChatBackend extends Cloudflare.DurableObjectNamespace<ChatBackend>()(
  "ChatBackend",
  Effect.gen(function* () {
    return Effect.gen(function* () {
      return {};
    });
  }),
) {}
```

A bare DO with an empty public API. We'll build it up step by step.

## Provide the persistence layer

```diff lang="typescript"
 import * as Cloudflare from "alchemy/Cloudflare";
 import * as Effect from "effect/Effect";
+import * as Layer from "effect/Layer";
+import { Persistence } from "effect/unstable/persistence";

 export default class ChatBackend extends Cloudflare.DurableObjectNamespace<ChatBackend>()(
   "ChatBackend",
   Effect.gen(function* () {
     return Effect.gen(function* () {
+      const persistence = yield* Persistence.layerResultPersisted({
+        storeId: "alchemy.chat",
+      }).pipe(Layer.provide(Cloudflare.DurableObjectChatPersistence));
       return {};
     });
   }),
 ) {}
```

`Persistence.layerResultPersisted({ storeId })` is the seam Effect AI
exposes for cached/replayable AI calls.
`Cloudflare.DurableObjectChatPersistence` provides the storage —
every entry lives in the DO's `state.storage` under the
`alchemy.chat:` key prefix.

## Add a `send` method

```diff lang="typescript"
+import { Chat, LanguageModel } from "effect/unstable/ai";

 // ...
       const persistence = yield* Persistence.layerResultPersisted({
         storeId: "alchemy.chat",
       }).pipe(Layer.provide(Cloudflare.DurableObjectChatPersistence));

-      return {};
+      return {
+        send: (threadId: string, prompt: string) =>
+          Effect.gen(function* () {
+            const chat = yield* Chat.fromPrompt({ prompt });
+            const response = yield* LanguageModel.generateText({ prompt });
+            return response.text;
+          }).pipe(Effect.provide(persistence)),
+      };
```

Each `send` call is wrapped in `Effect.provide(persistence)`, so the
chat machinery records the exchange in the DO's storage. The DO
instance *is* the conversation; calling `getByName(threadId)`
selects a thread.

## Multiple stores in the same DO

```typescript
const aiPersistence = yield* Persistence.layerResultPersisted({
  storeId: "alchemy.chat",
}).pipe(Layer.provide(Cloudflare.DurableObjectChatPersistence));

const auditPersistence = yield* Persistence.layerResultPersisted({
  storeId: "alchemy.audit",
}).pipe(Layer.provide(Cloudflare.DurableObjectChatPersistence));
```

Different `storeId`s namespace their keys with `${storeId}:`, so
they don't collide. Use this when one DO needs to keep, say, chat
history *and* an audit log in separate stores.

## Storage shape

Behind the `BackingPersistence` interface, each entry is stored as a
single struct value via `state.storage.put(key, value)`. DO storage
is SQLite-backed, so reads are O(log n) and bulk operations
(`getMany`, `setMany`) translate to one SQL query each.

:::caution[No TTL]
DO storage has no native TTL — the persistence layer ignores TTLs you
pass to `set`. If you need expiry, layer a TTL-aware backing store on
top, or run a periodic `clear`.
:::

## Recap

- `DurableObjectChatPersistence` is a `BackingPersistence` layer
  backed by `DurableObjectState.storage`.
- Provide it underneath
  `Persistence.layerResultPersisted({ storeId })` inside a DO's
  inner Effect; multiple `storeId`s coexist in one DO.
- Survives restarts, hibernation, and redeploys; storage is per-DO
  instance, so the natural unit of isolation is "one chat per
  `getByName(id)`".
